## The Problem: AI Model Collapse
As Large Language Models (LLMs) scrape the internet for training data, they increasingly ingest synthetic code generated by other AI models. This feedback loop degrades the mathematical distribution of the data, leading to **Model Collapse**. 

Current text-based watermarks (like hidden comments or variable renaming) fail because standard auto-formatters (e.g., `Black`, `Prettier`) instantly erase them. The industry requires an un-erasable fingerprint to filter human-written data from synthetic noise.

## The Solution
ASTrace Abandons text-based watermarks in favor of **Structural Transposition**. The engine compiles Python source code into an Abstract Syntax Tree (AST) and uses a SHA-256 hashed secret key to deterministically swap commutative math nodes (e.g., converting `A + B` to `B + A`). 

Because this fingerprint is baked into the mathematical skeleton of the program:
1. It is 100% immune to automated code formatters.
2. It executes perfectly without altering runtime logic.
3. It utilizes "Opaque Predicates" (e.g., `_wm_cache = (1*1)+0`) as padding to guarantee secure entropy for small files.
